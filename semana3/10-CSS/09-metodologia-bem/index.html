Metodología BEM

BEM es una metodología de nomenclatura para nombrar las clases de tus elementos HTML.

¿Cómo funciona BEM?

El objetivo de BEM es dar mucha más transparencia y claridad en tu estructura HTML y CSS.

BEM son tres siglas.

  • B de Bloque.
  • E de Elemento.
  • M de Modificador.

Aquí una explicación másdetallada sobre estos tres conceptos de BEM:

B

Bloque

Es el contenedor principal del componente y debe ser una entidad propia que no dependa de ningún otro bloque. Esto no quiere decir que no podamos tener un bloque dentro de otro, sino que estos serán independientes entre si, cada bloque tendra sus propios elementos con sus diferentes modificadores (bloque).

E

Elemento

Son las partes internas que conforman el componente y se usa para ir dividiendo el bloque en segmentos más pequeños, se debe colocar el nombre del bloque al que pertenece seguido por 2 guiones bajos (bloque__elemento).

M

Modificador

Son variaciones del bloque o del elemento y sirve para modificar algunas propiedades ya sea del bloque o algun elemento en particular, estos se usan agregando un doble guion medio justo después del nombre del elemento o bloque que se quiere modificar (bloque__elemento--modificador).

Así se vería el HTML:

        
          <div class="bloque">

            <div class="bloque__elemento">Elemento 1</div>

            <div class="bloque__elemento">Elemento 2</div>

            <div class="bloque__elemento bloque__elemento--modificador">Elemento 3</div>

          </div>

El CSS siguiendo BEM

        
          .bloque { color: inherit; }

          .bloque__elemento { color: inherit; }

          .bloque__elemento--modificador { color: inherit; }

Los nombres de las clases con convención BEM, pueden tener la siguiente sintaxis:

  • [bloque]
  • [bloque]__[elemento]
  • [bloque]--[modificador]
  • [elemento]--[modificador]
  • [bloque]__[elemento]--[modificador]
