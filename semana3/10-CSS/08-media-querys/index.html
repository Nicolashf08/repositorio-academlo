Media Querys

Una de las características más importantes de las hojas de estilos CSS es que permiten definir diferentes estilos para diferentes medios o dispositivos: pantallas, impresoras, móviles, proyectores, etc.

Los medios más utilizados actualmente son screen (para definir el aspecto de la página en pantalla) y print (para definir el aspecto de la página cuando se imprime).

@media

Las reglas @media son un tipo especial de regla CSS que permiten indicar de forma directa el medio o medios en los que se aplicarán los estilos incluidos en la regla. Para especificar el medio en el que se aplican los estilos, se incluye su nombre después de @media. Si los estilos se aplican a varios medios, se incluyen los nombres de todos los medios separados por comas.

  1. Keyword: Palabra reservada, @media.
  2. Media Type: Tipo de medio, print, screen, speach, etc.
  3. Operator: Operadores, and | or | not.
  4. Media Condition: Condición del medio, min-width, max-width, min-height, max-height, etc.
  5. Breakpoint: Valor de la condición, 320px, 576px, etc.
        
        @keword type operator (conditional: breakpoint) {

          selector {

            prop: value;

          }

        }

Ejemplos de media queries

Veamos un ejemplo clásico de media queries en el que definimos diferentes estilos dependiendo del dispositivo que estamos utilizando.

        
        @media screen and (min-width: 320px) {

          .small {

            font-size: 1.2rem;

            color: #333;

          }

        }

Actualmente podemos usar los media queries de esta forma, es totalmente valido.

        
        @media (max-width: 320px) {

          .small {

            font-size: 1.2rem;

            color: #333;

          }

        }

Mobile First

En 2015, Google anunció que por primera vez había recibido más búsquedas realizadas desde dispositivos móviles que de escritorio. Como consecuencia, la industria online ha puesto un mayor interés en el desarrollo de nuevos conceptos y estrategias de diseño web móvil. De esta forma, la metodología Mobile First ha revolucionado la creación de páginas al enfocarse primero en el diseño para terminales móviles pensando también en su usabilidad y rendimiento, para luego adaptarlas a la pantalla de un ordenador.

¿Qué diferencias existen entre Mobile First y Responsive?

A menudo existe cierta confusión entre estos dos términos, por ello, es necesario conocer en qué se diferencian y cuál es el planteamiento desarrollado por cada una de estas metodologías de diseño.

Por un lado encontramos el diseño web Responsive, el cual consiste en pensar y crear una web en formato para ordenadores o portátiles y, finalmente, adaptar todos los componentes a versión móvil.

En otra perspectiva encontramos el diseño Mobile First que trata de desarrollar el sitio web inicialmente en formato móvil y la adaptación se experimentará en el PC.

Obviamente, el diseño web tradicional para ordenadores y portátiles no se debe dejar de lado, pero quien quiera que su negocio llegue a una mayor cantidad de clientes potenciales, y por lo tanto generar un mayor tráfico a su página web, deberá concentrarse en el desarrollo de una estrategia de diseño web móvil. Empezar a trabajar usando la estrategia Mobile First, no representará en ningún momento un retroceso técnico. Gracias a su sencilla estructura, limitada a lo esencial, es posible adaptarla y añadirle nuevas características una vez haya sido terminada.
